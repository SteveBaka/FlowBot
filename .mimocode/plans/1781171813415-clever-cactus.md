# 计划：消息发送功能修复 + 设置开关

## 背景
应用运行正常但消息发送不可用，聊天框无响应。根因分析：
- `MessageInput` 组件 → `apiSendMessage` → POST `http://127.0.0.1:5031/api/v1/messages/send`
- HTTP API 服务要求 `httpApiEnabled: true` 且有活跃的 Hook 插件
- 当前无 Hook 插件激活时，API 返回 503 错误
- ChatPage 的 `onSendMessage` 只 `console.error` 了错误，没有给用户反馈

## 实施方案

### 1. 在设置中添加消息发送开关

**文件**: `src/services/config.ts`
- ConfigSchema 添加 `messageSendEnabled: boolean`（默认 `true`）
- 添加 getter/setter: `getMessageSendEnabled()` / `setMessageSendEnabled()`

**文件**: `src/pages/SettingsPage.tsx`
- 在消息推送开关附近添加"发送消息"开关
- 遵循已有的 toggle 模式（`switch` CSS class）

### 2. ChatPage 发送框跟随开关

**文件**: `src/pages/ChatPage.tsx`
- 读取 `messageSendEnabled` 配置
- `MessageInput` 的 `disabled` 属性绑定到此开关
- 当开关关闭时，发送按钮禁用、textarea 禁用

### 3. 修复消息发送错误反馈

**文件**: `src/pages/ChatPage.tsx` (line ~8322)
- `onSendMessage` 中添加用户可见的错误提示（`showMessage` 或 toast）
- 当前只 `console.error`，用户看不到失败原因

### 4. HTTP API 自动启动保障

**文件**: `electron/main.ts`
- 确保 `httpService.autoStart()` 在启动时被调用（已有 line 4442）
- 验证 `httpApiEnabled` 配置默认值

### 关键文件清单

| 文件 | 操作 | 说明 |
|------|------|------|
| `src/services/config.ts` | 修改 | 添加 messageSendEnabled 配置项 |
| `src/pages/SettingsPage.tsx` | 修改 | 添加发送消息开关 UI |
| `src/pages/ChatPage.tsx` | 修改 | 发送框跟随开关 + 错误反馈 |
| `electron/main.ts` | 检查 | 确保 HTTP API 自动启动 |

### 验证步骤
1. 构建并运行应用
2. 在设置中开启/关闭"发送消息"开关
3. 验证聊天页发送框根据开关状态启用/禁用
4. 开启状态下尝试发送消息，验证错误反馈
5. 关闭状态下验证发送按钮和输入框被禁用
